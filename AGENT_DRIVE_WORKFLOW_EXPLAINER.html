How this works: SGAI reads agents and model from GOAL.md frontmatter. It launches only the coordinator with opencode run --agent coordinator --model <model> [--variant <variant>]. The coordinator receives a prompt containing the allowed availableAgentsForDelegation and delegates specialist work through OpenCode subagent delegation — SGAI does not cycle through listed agents or assign per-agent models.

Blocking rules: coordinator is implicit and never appears in the delegatable list. stpa-analyst is retired — STPA remains a skill workflow, not a routable agent. Unlisted names and duplicate entries are also blocked. The pre-prompt filter audit captures all blocked candidates as evidence; they are never appended to the coordinator context.

OpenCode subagent delegation handles specialist agents (Go, React, general-purpose, etc.). SGAI does not need to route between them — the coordinator delegates directly through OpenCode's task/subagent system.
